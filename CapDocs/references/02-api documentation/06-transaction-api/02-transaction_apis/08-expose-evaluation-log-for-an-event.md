This API allows you to expose evaluation log for an event using Transaction Details (transactionId, transactionNumber).

# Prerequisites

-   Authentication: Basic or OAuth authentication
-   Default access group

# Resource information

|               |                                      |
| :------------ | :----------------------------------- |
| URI           | /v2/transactions/searchEvaluationLog |
| HTTP method   | POST                                 |
| Pagination    | NA                                   |
| Rate limit    | NA                                   |
| Batch support | NA                                   |

# API endpoint example

`https://eu.api.capillarytech.com/v2/transactions/searchEvaluationLog`

# Request Body parameter

| Parameter           | Data Type | Description                                                                                                                                                                                                                       |
| ------------------- | --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| search              | Object    | It is used to specify criteria for retrieving or filtering specific records, in this case by the transaction details (transactionId, transactionNumber).                                                                          |
| transactionNumber\* | String    | Unique transaction number which is used to fetch the evaluation log. You can get the Transaction Details (transactionId, transactionNumber). from this [API](https://docs.capillarytech.com/reference/get_transaction_detailsv1). |

```json Sampe POST body
{
  "search": {
    "transactionNumber": "num-1784"
  }
}
```

# Response body parameter

| Parameter      | Description                                                                              |
| -------------- | ---------------------------------------------------------------------------------------- |
| id             | Unique identifier for the transaction.                                                   |
| customerId     | ID of the customer associated with the transaction.                                      |
| date           | Date and time of the transaction in ISO 8601 format.                                     |
| eventName      | Name of the event associated with the transaction.                                       |
| hostName       | Name of the host that processed the transaction.                                         |
| orgId          | Organization ID associated with the transaction.                                         |
| requestId      | Unique identifier for the request associated with the transaction.                       |
| transactionId  | Unique identifier for the transaction.                                                   |
| uniqueId       | A unique identifier generated for this transaction.                                      |
| evaluationTree | HTML structure representing the evaluation tree of rules applied during the transaction. |
| effects        | Promotion ID associated with the voucher issue.                                          |
